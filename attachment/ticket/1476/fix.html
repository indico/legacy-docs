14=== Instalação da fonte

16==== Q: Por que eu recebo NameError: 'log' nome global não está definido quando tento usar setup.py?
17
18{{{
19../trunk/$ python setup.py egg_info
20running egg_info
21...
22  File "build/bdist.linux-x86_64/egg/setuptools/svn_util.py", line 55, in get_sections
23NameError: global name 'log' is not defined
24}}}
25
26**A:** Você é mais provável trabalhar a partir de uma subversão ** 1,7 ** checkout e você estiver usando um velho demais `pacote` setuptools (por exemplo, `/ setuptools-0.7a1dev_r66608`).
27
28Veja # 7598. Upgrade para uma setuptools mais novos, de downgrade para svn 1.6.x, use distribute.
29
30==== Q: Porque é que a instalação do `distribute` não `com ValueError: A setuptools 0,7 série não pode ser instalado com distribute`.?
31
32Você tentou instalar o `` distribute como aconselhado em outro lugar:
33{{{
34$ curl http://python-distribute.org/distribute_setup.py | python
35}}}
36mas você é saudado com:
37{{{
38ValueError: A 0.7-series setuptools cannot be installed with distribute. Found one at /packages/trac/virtualenv-0.13/lib/python2.7/site-packages/setuptools-0.7a1dev_r66608-py2.7.egg
39}}}
40
41**A:**Não desista, é apenas `` setuptools dando-lhe uma última luta.
42
43Você pode remover os setuptools ofensivos. egg ... e se mantém re-aparecer, você tem instalado não só no virtualenv mas também na instalação pai Python (aquele em que você instalou o virtualenv em si). Se este for o caso, retire-o de lá também, a instalação distribuem agora deve funcionar.
44
45==== Q: Por que acidentes Apache (500) com um ZipImportError `no log de erro?
46
47{{{
48[Tue Aug 07 09:08:26 2012] [error] [client 172.25.16.59] mod_wsgi (pid=16444): Exception occurred processing WSGI script '/packages/trac/virtualenv-0.13/bin/trac.wsgi'., referer: http://dserver/trac/bct/timeline
49...
50[Tue Aug 07 09:08:26 2012] [error] [client 172.25.16.59]   File "build/bdist.linux-x86_64/egg/pkg_resources.py", line 1197, in _get, referer: xxx/trac/bct/timeline
51[Tue Aug 07 09:08:26 2012] [error] [client 172.25.16.59] ZipImportError: bad local file header in /packages/trac/virtualenv-0.13/lib/python2.7/site-packages/Genshi-0.6-py2.7.egg, referer: xxx/trac/bct/timeline
52}}}
53
54**A:** Você certamente se esqueceu de parar e reiniciar o servidor durante uma atualização. O mecanismo importador zip mantém um cache interno, que tem problemas para detectar arquivos substituídos, por isso é necessário reiniciar (ver ZipImportError TracUpgrade #). Faça isso agora.
55
56==== Q: Trac / about diz 1.0, mas o estilo parece pré-1.0
57
58**A:** Você provavelmente só precisa convencer o seu navegador que os arquivos CSS mudaram. Tente "recarregar a força" (CTRL + `R` ou `CTRL + Shift + R`, dependendo do seu navegador). Este é um problema comum (ver, por exemplo, # 10797).
59
60Se isso não funcionar, procure na sua configuração do servidor para ver onde os arquivos são cromo deve ser servido a partir de (`alias` ou `AliasMatch directive`), e talvez você vai ver que você está servindo-os como estático arquivos de um local diferente:
61{{{
62Alias /trac/bct/chrome /packages/trac/virtualenv-0.13/share/htdocs
63}}}
64Neste caso, não se esqueça de:
65{{{
66$ trac-admin <yourenv> deploy /packages/trac/virtualenv-0.13/share
67}}}
68Veja TracUpgrade # a5.Refreshstaticresources.
