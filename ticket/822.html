#822 closed task (wontfix)

Participants: allow to add all users from a list

Reported by: jbenito Owned by:
Priority: normal Milestone: v2.5
Component: General Version:
Keywords: Cc:

Description

On another note, it would be useful to in participants to lectures
to be possible to include a group as a participant. This way one
wouldn't have to enter them one by one. At the moment this does not
seem possible. Even better: choosing a group would add to
participants a list with their members. Then one could delete
anybody if this was necessary.

Change History (2)

comment:1 Changed 3 years ago by jbenito

  • Milestone changed from v1.xx to v1.7
  • Priority changed from very low to normal

Beware that in order to do this, we need a service that can return all the members of a group based on the user logged in. This means, that the members will be listed only if the user has rights to access the member list. Otherwise, it could be a security problem.
